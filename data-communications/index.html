CH 03 물리층 개요
정현파 & 신호 기초
s = A · sin(2πft + φ)
A최대 진폭 (신호 세기) f주파수 (Hz) — 높을수록 파형 촘촘 → 데이터 전송↑ T주기 = 1/f (한 번 반복 시간) φ위상 (파형 시작 위치) t현재 시각 (흐르는 시간)
λ = c/f    c = 3×10⁸ m/s
BW (대역폭) = f_max − f_min
아날로그 vs 디지털 · 전압 준위
아날로그디지털
장점자연 신호 그대로, 변환손실 없음잡음에 강함, 암호화 용이, 복사 품질 유지
단점잡음 취약, 복사 품질 저하, 암호화 어려움변환 시 정보 손실, 대역폭 많이 필요
단극형 (Unipolar)
+V / 0V · 직류 성분 심함
극형 (Polar)
+V / -V · 양음 균형 → 직류 성분 적음
디지털 신호 = 이론상 무한 대역폭 (수직 전환 → 무한 고주파 필요)
전송률 한계 공식
나이퀴스트 — 잡음 없는 채널
Bit Rate = 2 × BW × log₂L
BW대역폭 (Hz) L신호 준위 수
섀논 — 잡음 있는 채널
C = BW × log₂(1 + SNR)
SNR신호 대 잡음비
SNR_dB = 10 × log₁₀(SNR)
감쇠 dB = 10 × log₁₀(P₂/P₁)
1+SNR 이유: SNR=0이면 log₂(0)=음의 무한대 → C=0으로 자연스럽게 처리
실제 설계: 섀논으로 이론 상한 → 나이퀴스트로 신호 준위 수 계산
CH 04 디지털 전송
회선 부호화 기본 개념 & 문제점
S = N / r
S신호 전송률 (baud) — 1초당 신호 요소 수 N데이터 전송률 (bps) — 1초당 비트 수 r신호 요소당 데이터 요소 수
문제점원인해결
기준선 표류연속 0/1 → 수신측 기준선 이동맨체스터
직류 성분저주파 → 전선·변압기 통과 불가극형 방식
자기 동기화클럭 불일치 → 비트 오독맨체스터
회선 부호화 파형 비교 (비트: 1 0 1 1 0)
맨체스터 (절대적)
반전 방향 자체가 비트
극성 뒤집히면 → 오독
차분 맨체스터 (상대적)
전이 유무만 봄
극성 뒤집혀도 → 정확히 읽음
부호화 방식 비교
방식대역폭동기화잡음사용
NRZ1배약함보통USB, 시리얼
RZ2배좋음보통광섬유 일부
맨체스터2배좋음보통이더넷, RFID
차분맨체스터2배좋음강함토큰링
RZ 대역폭 2배 이유: 비트당 변화가 NRZ 1번 → RZ 2번 → 2배 빠른 주파수 필요
다준위 방식 — 4D-PAM5
PCM (아날로그→디지털) — 표양부
  • 표본화 (Sampling) — 일정 간격으로 진폭 측정
    표본화율 ≥ 최대주파수 × 2 · 예) 음성 4kHz → 8,000 samples/s
  • 양자화 (Quantizing) — 실수값을 정수로 근사
    양자화 잡음 발생 (근사 오차)
  • 부호화 (Encoding) — n비트 코드로 변환
Δ = (V_max − V_min) / L
Δ양자화 간격 (구간 크기) — L↑ → Δ↓ → 잡음 감소 V_max신호 최대 전압 (maximum voltage) V_min신호 최소 전압 (minimum voltage) L양자화 준위 수 (levels)
Bit rate = f_s × n_b
f_s1초당 표본화 횟수 (sampling frequency) n_b표본당 비트 수 (number of bits per sample)
예) 음성: 8,000 × 8bit = 64,000 bps
PCM + 광대역: 아날로그→[PCM]→디지털→[광대역 변조]→전송 · 예) VoIP
