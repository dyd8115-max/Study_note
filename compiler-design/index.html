SCENE 01 어휘분석이란? — 스캐너의 역할
컴파일러 속 어휘분석기의 위치
핵심 — Scanner는 토큰을 한꺼번에 다 찾는 게 아니다. Parser가 getToken()을 호출할 때마다 하나씩 꺼내서 넘긴다.
어휘분석기의 기능
  • 토큰 전달Parser 요청 시 getToken() 호출 → 토큰 반환
  • 공백·주석 제거white space, comment 무시하고 skip
  • 기호표 구성식별자를 symbol table에 등록
getToken() 동작
TokenType getToken(void);

// 입력 버퍼:
// a [ i n d e x ] = 4 + 2
// ↑
// 첫 호출 → return ID("a")

// Token의 type : ID
// Token의 value: "a"
Parser가 부를 때마다 버퍼에서 다음 토큰을 하나 읽어 반환한다.
핵심 용어 3가지
용어정의예시
Token Terminal 기호로만 구성된 문법적 최소 단위 if, 123, +, myVar
Pattern 토큰의 특징을 표현하는 규칙 → 정규 표현(RE) letter(letter|digit)*
Lexeme Pattern과 매칭되는 실제 문자열 count, total2, _var
비유 — 패턴은 "식별자란 알파벳으로 시작하는 문자열"이라는 규칙, 렉심은 소스코드 속 실제 문자열 (myVar, count).
SCENE 02 Delimiter & Lookahead — 경계를 어떻게 처리하나
Delimiter (구분기호)
정의현재 토큰이 여기까지임을 알려주는 경계
중요구분기호는 현재 토큰에 포함 안 됨
xtemp=ytemp
→ =이 delimiter, xtemp가 토큰
while x
→ 공백(blank)이 delimiter
do/* */if
→ 주석(comment)이 delimiter
whitespace = (newline | blank | tab | comment)+
Lookahead (미리 보기)
문제 — delimiter가 다음 토큰의 일부일 수 있다
입력delimiter다음 토큰
xtemp=ytemp== 자체
x+=2;++=
해결: 읽은 문자를 버리지 말고
• 현재 토큰 일부 → 지금 처리
• 아니면 → 버퍼에 돌려두고 다음 토큰 때 사용
비유 — 신호등 노란불: 읽었지만 아직 처리할지 결정 안 한 상태
SCENE 03 삼위일체 — RE · FA · RG는 같은 표현력
정규 표현 = 유한 오토마타 = 정규 문법
어휘분석기 설계 흐름: 토큰 패턴 → 정규 표현 → NFA → DFA → 최소화된 DFA → 코드
Chomsky 형식 문법 분류
Type문법언어인식기컴파일러
3정규 문법 RG정규 언어 RL유한 오토마타 FA스캐너
2문맥자유 CFG문맥자유 CFL푸시다운 오토마타파서
1문맥인식 CSG문맥인식 CSL선형한계 오토마타—
0무제약 UG재귀열거튜링 기계—
자주 나오는 언어 분류: aⁿbⁿ → CFL / aⁿbⁿcⁿ → CSL / 회문(palindrome) → CFL / 괄호 → CFL
SCENE 04 FA → 정규 문법 변환
변환 규칙
VN = Q상태 → Nonterminal
VT = Σ입력 알파벳 → Terminal
S = q₀시작 상태 → 시작 기호
δ(q,a)=r → q → ar
q ∈ F → q → ε
결과물은 Right-linear Grammar (A → aB 형태)
핵심 공식 — RE 방정식의 해
X = αX + β → X = α*β
증명:
X = α(α*β) + β
  = α⁺β + β
  = (α⁺ + ε)β
  = α*β  ∎
모든 변환 문제가 이 꼴로 귀결됨. 외워야 함.
예제 — FA → RE 변환 (L = (a+b)*abb)
D = ε,   C = bD = b,   B = bC = bb
A = aA + bA + aB = (a+b)A + abb
  →  A = (a+b)* abb

∴ L(M) = (a+b)*abb
정규 표현 대수 법칙
법칙식
결합(α+β)+γ = α+(β+γ)
교환α+β = β+α
분배α(β+γ) = αβ+αγ
항등α+α = α, εα = α
클리니ε+αα* = α*
이중클리니(α*)* = α*
기호 의미
+ 또는 | → OR (선택)
· → 연결 (concatenate)
* → 0회 이상 반복 (Kleene star)
+ 위첨자 → 1회 이상 반복
ε vs ∅
{ε} = ε 원소 1개 있는 집합
∅ = 원소가 하나도 없는 공집합
→ 다르다!
SCENE 05 변환 예제 모음 — 정규 문법 → 정규 표현
예제 1 — L(G) = (aa+b)*ab(a+b)*
정규 문법
S → aA | bS
A → aS | bB
B → aB | bB | ε
풀이
B = (a+b)B + ε  →  B = (a+b)*
A = aS + b(a+b)*
S = aA + bS = a(aS+b(a+b)*) + bS
  = aaS + ab(a+b)* + bS
  = (aa+b)S + ab(a+b)*
  →  S = (aa+b)* ab(a+b)*
예제 2 — L(G) = b*a(a+b)*b
정규 문법
S → aA | bS
A → aA | bA | b
풀이
A = (a+b)A + b  →  A = (a+b)*b
S = a(a+b)*b + bS
  = bS + a(a+b)*b
  →  S = b*a(a+b)*b
예제 4 — Unreachable State 제거 → L(G) = (ab+ba)*
도달 불가 전이 제거 후
A → aB | bC | ε
B → bA   (aD 제거)
C → aA   (bD 제거)
풀이
B = bA, C = aA
A = aB + bC + ε
  = a(bA) + b(aA) + ε
  = (ab+ba)A + ε
  →  A = (ab+ba)*
Unreachable state — 시작 상태에서 절대 도달할 수 없는 상태. 해당 전이는 정규 문법 변환 시 무시한다.
SCENE 06 토큰 인식 — 정수 & 식별자
정수(Integer) 인식 — d+
정규 문법:
S → dC
C → dC | ε

정규 표현:
C = dC + ε  →  C = d*
S = dC = dd*

∴ L(G) = d+
식별자(Identifier) 인식 — (l+_)(l+d+_)*
정규 문법:
S → lA | _A
A → lA | dA | _A | ε
정규 표현:
A = (l+d+_)A+ε → (l+d+_)*
S = (l+_)A

∴ L(G) = (l+_)(l+d+_)*
가능: sum, _count, myVar2    불가: 2user(숫자 시작), 123
