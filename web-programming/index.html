Web 07 Asynchronous Programming
동기 vs 비동기
동기
• 순서대로 실행
• 대기 필수
• ❌ 블로킹
비동기
• 백그라운드 실행
• 기다리지 않음
• ✅ 논블로킹
Promise 3가지 상태
Pending → Fulfilled (.then()) ↘ Rejected (.catch())
상태메서드설명
Fulfilled.then()resolve() 호출
Rejected.catch()reject() 호출
항상.finally()성공/실패 무관
⚠️ Promise는 한 번만 변경 · resolve 후 reject 호출은 무시
Fetch API
fetch(url) .then(res => res.json()) // Promise 반환! .then(data => console.log(data)) .catch(err => console.log(err))
Promise 반환: 비동기 네트워크
response.json(): 바디 크기 크면 Promise
async/await
async function getData() { try { const res = await fetch(url); const data = await res.json(); return data; } catch(error) { console.log(error); } }
async: 함수를 비동기로 · 항상 Promise
await: Promise settle 대기 · async 내에서만
💡 병렬: await Promise.all([p1, p2, p3])
